## 三、各层详细设计 & 实施任务

### ③ AI 决策层 — Scheduler Agent

**核心职责**: 每 N 分钟跑一次, 产出"未来 1 小时发布计划".

**输入**:
- `drama_banner_tasks` (候选剧池 + 30 天分佣)
- `device_accounts` (在线账号 + 最近发布次数)
- `mcn_member_snapshots` (账号最近收益 + 风控信号)
- `publish_results` (历史成败 + 播放量)

**输出** (写进 `task_queue`):
```python
{
  "account_id": 3,
  "drama_name": "陆总今天要离婚",
  "drama_link_url": "...",          # 从 drama_links 挑
  "process_variant": "kirin_mode6", # 去重方式
  "scheduled_at": "2026-04-19 14:30",
  "priority": 80,
  "experiment_id": None,             # 如果是 A/B 测试的话
}
```

**3 种策略模式** (配置项 `selector.strategy`):
- `top_by_income` — 永远挑 TOP 5 最赚钱 (保守)
- `top_weighted_random` — 从 TOP 50 按收益加权抽 (**推荐**, 已是默认值)
- `amplify_test` — 放大测试: 同一剧 3-5 个视频版本, 不同账号, 对比播放量

**放大测试 (Amplify Test) 子流程**:
1. 选 1 条 TOP 剧 (e.g. 少年叶飞鸿 ¥11,916)
2. 从该剧 10 个不同视频地址各下载 1 个
3. 分配到 5 个账号发 (每账号 2 个变体)
4. 48h 后对比每个变体的播放量 + 收益贡献
5. 把赢家视频特征喂回 Scheduler (以后优先挑相似特征)

### ④ 执行器 — Account Executor

**核心职责**: 从 `task_queue` 消费 PENDING 任务, 执行完整流水线, 写回结果.

**并发模型** (`concurrency.mode` 配置):
- `link` (链接级): 所有账号发剧 1, 全完成, 再发剧 2
- `account` (账号级): A 账号独立跑完所有剧, 不等其他
- `hybrid`: 每 N 个账号一批 (184 默认)

**状态机**:
```
queued → downloading → download_failed (retry or dead)
                     → processing     → process_failed
                                      → deduping       → dedupe_failed
                                                       → publishing
                                                          → published ✓
                                                          → submit_rejected
                                                          → verify_pending
                                                          → verified ✓
```

```python
def download_drama(drama_name, attempts_per_url=2):
    urls = db.query_drama_urls(drama_name, order_by="use_count ASC")
    for i, url in enumerate(urls):
        for attempt in range(attempts_per_url):
            try:
                path = download_one(url, timeout=60)
                if verify_mp4(path):  # ffmpeg -i 检查完整性
```

```python
    new_filters = llm.search_and_synthesize(prompt, web_search=True)
    
    # 2. 生成新 recipe, 加进候选池 (初始 5% 权重 = canary)
    for recipe in new_filters:
        db.insert_recipe(recipe, weight=0.05, status='experimental')
    
    # 3. 一周后 review, 表现好的提权, 差的淘汰
```
