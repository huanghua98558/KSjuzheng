### 2.2 完整数据流 (一个发布请求)

```
客户 PyQt6 点击"立即发布"按钮
  ↓ Signal
PublishService.start_batch(selected_dramas, selected_accounts)
  ↓ httpx.post
POST /api/client/publish/batch        ← 客户端视图 API
  Authorization: Bearer <JWT>
  X-License-Key: <卡密>
  ↓ router → facade
client_publish_facade.create_batch(...)   ← 客户端专属 facade
  ↓ 内部转化 (隐藏技术细节)
对客户: "24 条任务已加入队列, 预计 15 分钟完成"
对内部: 创建 24 个 task_queue 行, recipe=根据账号 tier 自动选
  ↓
core.executor.pipeline.run_publish_pipeline (最终执行)
  ↓ 返回 Envelope
{ok: true, data: {batch_id, estimated_minutes: 15, total: 24}, meta: {...}}
  ↓ UI
显示"发布任务已启动, 预计 15 分钟" + 跳转到"发布结果"页
```

### 2.3 FastAPI 改造目标重定义

**不是**: 把现有 179 端点改造成给运营端用得更爽
**是**: **为用户端客户端设计一套干净的高级抽象 API**, 同时兼容运营端继续用.

新端点分两类:

```
/api/client/*       ← 客户端视图 API (新设计, 高级抽象)
  /api/client/overview         首页汇总
  /api/client/accounts         账号列表 (不带 tier/memory/决策等内部字段)
  /api/client/dramas           剧榜
  /api/client/publish/batch    批量发布 (不暴露 recipe/sig)
  /api/client/revenue          收益统计
  /api/client/exceptions       异常 (高级抽象, 不暴露 breaker/task_id)
  /api/client/ai/advice        AI 建议 (人话, 不暴露 LLM provider)
  /api/client/autopilot        AutoPilot 状态 (运行/暂停两态, 不暴露 cycles)
  ...

/api/admin/*        ← 运营端 API (保留现有 179 端点)
  所有现有 /api/* 移入此前缀, 或加独立 middleware 标识
  只能 dev token 访问
```

**隔离原则**:
- 客户端 JWT 只能调 `/api/client/*`, 调 `/api/admin/*` 返 403
- 运营端 dev token 两边都能调 (调试用)

### 7.2 错误码体系 (用户友好)

| 错误码 | HTTP | 用户可读文案 | 后端用途 |
|---|---|---|---|
| `AUTH_401` | 401 | 登录已过期, 请重新登录 | JWT 过期 |
| `AUTH_402` | 402 | 您的套餐已到期, 请前往订阅中心续费 | 卡密过期 |
| `AUTH_403` | 403 | 此功能需升级到高级版 / 团队版 | feature 不匹配 |
| `AUTH_423` | 423 | 账号已被锁定, 请联系客服 | 风控触发 |
| `VALIDATION_422` | 422 | 请求参数不正确, 请检查输入 | Pydantic 校验 |
| `RESOURCE_404` | 404 | 请求的资源不存在 | 资源缺失 |
| `CONFLICT_409` | 409 | 操作冲突, 请刷新后重试 | 乐观锁 |
| `RATE_LIMIT_429` | 429 | 请求过于频繁, 请稍后再试 | 限流 |
| `INTERNAL_500` | 500 | 服务器内部错误, 请稍后重试 | 未捕获异常 |
| `UPSTREAM_502` | 502 | 上游服务暂时不可用 | MCN/快手故障 |
| `BUSINESS_NO_URL` | 200 | 该剧暂无可用素材, 请稍后再试 | 无 URL |
| `BUSINESS_BLACKLIST` | 200 | 该剧已被屏蔽, 无法发布 | 黑名单命中 |
| `BUSINESS_MCN_80004` | 200 | 账号暂时无法发布该剧, 系统将自动切换 | 80004 |

**关键**: 错误 `message` 对照 v3 §18.3 对照表, **禁止** 出现 `sqlite3.OperationalError`, `cookie invalid`, `breaker triggered` 等技术原文.

### 7.3 两种认证 (关键区分)

```
用户端 (PyQt6 exe):
  Authorization: Bearer <JWT>
  X-License-Key: <卡密>                          # 每次携带, 后端校验
  X-Device-Fingerprint: <硬件指纹>              # 绑定机器

  只能访问 /api/client/*
  调 /api/admin/* → 403

运营端 (我们自己 Streamlit / web/):
  Authorization: Bearer <dev-token>             # 开发 token, 环境变量
  或 X-Dev-Token: <token>

  两边都能访问 (/api/client/* + /api/admin/*)

中间件 dependencies.py:
  jwt_license_auth  = Depends(verify_jwt_and_license)   # 客户端用
  dev_auth          = Depends(verify_dev_token)         # 运营端用
  auth_any          = Depends(verify_either)            # 调试用
```

### 7.4 客户端视图 API 设计原则

**核心原则**: 返回"高级抽象 + 人话", **不返回**内部技术字段.

对比示例 — `/api/client/exceptions` vs `/api/admin/incidents`:

```json
// ❌ 运营端 /api/admin/incidents (展示内部细节 — 给我们自己看)
{
  "id": 42, "severity": "critical",
  "type": "circuit_breaker_opened",
  "context": {
    "breaker_name": "mcn_relay_submit",
    "consec_fails": 7,
    "threshold": 5,
    "last_error": "sig3 reject, result=109"
  },
  "task_id": "task_abc123", "recipe": "kirin_mode6", "retry_count": 3
}

// ✅ 客户端 /api/client/exceptions (高级抽象 — 给客户看)
{
  "id": "exc_001",                            // 匿名 ID
  "severity": "high",
  "type": "publish_issue",
  "affected_object": "账号 @诗草莓酱",
  "description": "该账号最近 1 小时有 7 次发布失败, 系统已自动暂停",
  "suggestion": "建议: 检查账号登录状态",
  "suggested_action": "relogin",              // 枚举: relogin | retry | skip
  "created_at": "2026-04-25T14:30:00+08:00"
}
```

**字段过滤原则** (客户端 API **绝不**返回):
