## 一、产品定位

### 1.1 产品本质
**不是**: 内部工具 / 数据分析平台 / 运营后台
**是**: **给做快手短剧的个人 / 小工作室用的一键自动化 Windows 软件**

对标参考: KS184 (卖 ¥2000-5000 买断) / 其他抖音矩阵工具

### 1.2 核心价值主张

```
"你装了就能用, 我用 6 个月挣了 XX 万, 现在分享给你"

差异化 vs KS184:
  ✅ 创始人在用 (非纯卖软件)
  ✅ 数据驱动的 AI 决策 (非拍脑袋规则)
  ✅ 爆款雷达预判 (非追随数据)
  ✅ 持续服务 (订阅制, 规则/作者池/协议跟进云端推送)
```

### 1.3 目标用户

**用户画像**:
- 25-40 岁, 做快手短剧 CPS 分成或视频发布
- 单人或小团队 (1-5 人)
- 有 10-500 个快手账号
- 技术水平: 会用 Windows 软件, 不会 Python
- 痛点: 手动发布太累 / 不知道跟哪部剧 / 账号越多越乱

**预期付费用户**:
- Year 1: 50-200 付费用户
- Year 2: 500-1000
- Year 3: 2000-5000

## 四、技术架构决策

### 4.1 技术栈 (按 Phase)

| 阶段 | 客户端 | 服务端 | 加固 |
|---|---|---|---|
| **Phase 1 自用** | Python + Streamlit (现状) | 无 | 无 |
| **Phase 2 MVP** | Python + Streamlit | FastAPI (初版) | 无 |
| **Phase 3 对外 Beta** | **PyQt + Python** | FastAPI + Redis | 基础 Nuitka |
| **Phase 4 正式版** | PyQt + Python + Rust .pyd | FastAPI + PostgreSQL | **Nuitka+Rust+VMProtect** |
| **Phase 5 规模化** | 同上 | 迁 Go (高并发) | + Themida + 水印 |

### 4.2 关键技术决策

#### 客户端 UI: **PyQt6** (不是 Streamlit)

```
理由:
  ✅ 真原生 Windows 窗口 (对标 KS184)
  ✅ 启动 1-2 秒 (Streamlit 要 5-10 秒)
  ✅ 离线能用 (Streamlit 必须起 web 服务)
  ✅ 便于嵌 WebView / 反调试 hook
  ✅ 更难被逆向 (Streamlit 的 HTML/JS 暴露结构)

代价:
  ⚠️ 学习曲线 2-4 周
  ⚠️ 18 页 Streamlit 要全部重做 PyQt (~2 月工期)
  
时机: Phase 3 开始做, Phase 4 完成
```

#### 服务端框架: **FastAPI** 起步, Go 规模化

```
Phase 2-4: FastAPI
  + 和现有 Python 代码同语言, 迁移零摩擦
  + 生态成熟, 文档多
  + 异步性能够用 (~10k QPS 单机)

Phase 5+ (月付费用户 > 500): 迁 Go
  + 并发 100k+ 连接 (服务海量客户端心跳)
  + 镜像小, K8s 友好
  + 业务代码已稳定, 迁移风险低
```

#### 数据库: **PostgreSQL** + Redis

```
PostgreSQL (用户/订单/卡密/作者池):
  - 成熟的关系型, 事务强
  - 支持 JSONB (灵活 schema)
  - RDS 托管免运维

Redis (JWT/心跳/短期 token):
  - 毫秒级过期
  - 支持 pub/sub (实时推送爆款)

客户端 SQLite → SQLCipher:
  - 本地缓存, 加密
  - 密钥服务端下发, 只在内存
  - 杀进程 = 数据无法读
```

#### 客户端加固: **5 层叠加**

```
Layer 1: Nuitka 编译
  Python → C → 原生 exe
  (PyInstaller 的 .pyc 一分钟解, 不用)

Layer 2: 关键模块 Rust 化
  license_core.pyd  (授权校验)
  crypto_core.pyd   (加解密/签名)
  anti_tamper.pyd   (反调试/完整性)
  net_core.pyd      (HTTPS + 证书固定)

Layer 3: VMProtect 虚拟化
  标记关键函数 (license_check / fingerprint / 签名)
  翻译成自定义 VM 字节码, 极难逆

Layer 4: Themida 整体壳
  反调试 / 反 VM / 代码变形
  最外层防护

Layer 5: 资源加密 + 水印
  内嵌字符串/配置全 AES 加密
  每卡密的 exe 二进制不同 (UUID)
  泄露可追溯
```

### 4.3 自用 vs 对外 双构建

```
dev 构建 (自用):
  - 纯 Python, 零加固
  - 详细日志 / 热重载 / 直连 DB
  - 启动 <3s
  - make dev

prod 构建 (对外):
  - 全套加固
  - 日志加密 / 禁 debug / SQLCipher
  - 启动 5-10s
  - make prod

单一代码库, CI 两条 pipeline
```

## 六、安全防护分层 (对标火视界)

### 6.1 威胁模型

```
攻击者类型:
  L1 脚本小子     普通用户装软件瞎点 (无意)
  L2 破解新手     用现成工具 (Nuitka Decompiler / PyArmor Unpacker)
  L3 中级逆向     用 IDA / x64dbg / Frida 基础操作
  L4 高级逆向     有 Rust 反汇编经验, 能过 VMProtect
  L5 团队攻击     多人协作, 分析 1 月以上
  
我们目标: 
  L1/L2 100% 挡 (零成本)
  L3    挡 1 周+
  L4    挡 3 周+ (单人工时)
  L5    承认会被破 (关键: 服务端, 无算法可偷)
```

### 6.2 五层防护叠加

```
Layer 1: 服务端护城河 (最重要!)
  - 业务算法在云 (客户端即使被破也没用)
  - 数据按需下发 (用完即焚)
  - 每 5min 心跳验证

Layer 2: 客户端代码混淆
  - Nuitka 编译 (非 PyInstaller)
  - 关键模块 Rust .pyd (license / crypto / anti_tamper / net)
  - 字符串/配置 AES 加密

Layer 3: 虚拟化保护
  - VMProtect 虚拟化关键函数
  - 每 2 周换 VM 配置 (破解永远滞后)

Layer 4: 运行时检测
  - 反调试 (IsDebuggerPresent / PEB.BeingDebugged / RDTSC 时间差)
  - 反 Hook (检测 ntdll API 首字节是否 jmp)
  - 反 VM (VMware/VBox 特征检测)
  - 完整性校验 (exe .text 段 SHA256)
  
  触发后: 延迟 30s 崩溃 + 静默上报 (不弹窗)

Layer 5: 运营层
  - 每 2 周迭代 (破解永不完整)
  - 水印追溯 (每卡密独立 UUID)
  - 异常风控 (同指纹多卡密 / 地理跳变 / 调用模式异常)
```

### 6.3 关键对抗细节

```
❌ 错误做法:
  - 检测到破解立即弹窗 "盗版警告"
    → 攻击者立刻知道哪层触发, 针对性绕
  - 关键数据缓存到本地
    → dump 内存就能拿
  - 算法散在客户端
    → 破解一次, 永远能用

✅ 正确做法:
  - 检测到破解静默上报, 延迟 30s 崩溃 (假装系统问题)
    → 攻击者以为自己代码 bug, 绕半天
  - 数据按需拉, 用完即焚
    → 破解了也拿不到全量
  - 算法在服务端
    → 破解客户端 = 破解了个空壳
  - 每 2 周更新
    → 永远滞后 = 不值得破
```
