# 短剧自动剪辑系统完整开发文档

## 1. 文档定位

本文档面向研发实施，目标是将“完整长剧自动分析 -> 自动生成主力素材与辅助测试素材”的讨论内容，沉淀为一份可开发、可排期、可验收的完整开发文档。

本系统服务于“短剧挂载转化”场景，不是纯剪辑软件，也不是纯 CTR 广告素材生成器。系统的核心价值在于：

- 一次完整分析，建立剧情资产库
- 基于剧情资产库，多次轻编排，生成多个不同方向的素材
- 以 5 到 8 分钟主力素材承担承接转化
- 以 60 到 90 秒辅助素材承担卖点测试

本文档默认项目根目录为 `D:\ks_automation`。

---

## 2. 产品目标

### 2.1 核心目标

输入：

- 一部完整短剧母版，通常为 60 到 120 分钟
- 可选外挂字幕
- 可选剧本或剧情简介
- 可选运营偏好，如“更爽”“更虐”“更悬念”

输出：

- 主力素材 A：主线承接版，5 到 8 分钟
- 主力素材 B：主题强化版，5 到 8 分钟
- 辅助素材 1 到 6：60 到 90 秒
- 对应时间轴、字幕、标题建议、封面建议

### 2.2 非目标

本期不追求：

- 一步到位达到资深人工剪辑师成片质量
- 完全零人工介入
- 支持所有类型长视频
- 支持复杂综艺、纪录片、电影级叙事理解

### 2.3 成功标准

V1 达标条件：

- 一部整剧可稳定产出 2 条主力素材与 6 条辅助素材
- 主力 A 与主力 B 存在明确差异，不是简单换顺序
- 辅助素材具备清晰测试目标，不是同一爆点重复裁剪
- 人工复核时间控制在 10 到 20 分钟
- 任意任务失败后可从中间阶段恢复

## 3. 总体系统设计

### 3.1 设计原则

- 重分析只做一次，轻编排可多次执行
- 每一层都必须有可视化中间结果，方便排查与人工干预
- 主力素材与辅助素材从同一剧情资产库生成
- 模板驱动优先于自由生成
- 去重和多样性是核心约束，不是附加功能

### 3.2 逻辑分层

系统分为 6 层：

1. 接入层 ingest
2. 解析层 parse
3. 理解层 understand
4. 评分层 score
5. 编排层 assemble
6. 导出层 render

### 3.3 典型链路

```text
上传长剧
-> 视频标准化
-> ASR / 镜头切分 / OCR / 音频特征
-> 剧情段切分与理解
-> 剧情资产库
-> 主力模板 / 辅助模板编排
-> 时间轴导出
-> 人工快审
-> 生成可发布素材包
```

## 4. 业务术语定义

### 4.1 剧情资产库

指针对一部完整短剧，通过一次完整分析得到的结构化数据集合，包括：

- 台词时间轴
- 镜头时间轴
- 剧情段
- 冲突线
- 高光点
- 可做钩子的片段
- 可做悬念断点的片段

### 4.2 主力素材

5 到 8 分钟，用于承接转化、推动挂载点击的主素材。

### 4.3 辅助素材

60 到 90 秒，用于测试不同卖点、不同情绪、不同断点表现的测试型素材。

### 4.4 主题线

可运营的剧情线，例如：

- 女主逆袭线
- 婚姻冲突线
- 追妻火葬场线
- 豪门身份线
- 婆媳对立线

### 4.5 断点

视频结束时故意停在“未完成但极想继续看”的剧情节点，用于挂载转化。

## 7. 环节实施方案

本节按 7 个环节逐一说明：

- 输入
- 输出
- 失败兜底
- 人工介入点
- 开发重点

### 7.1 环节一：任务创建与素材接入

#### 输入

- 原始长视频文件
- 可选外挂字幕
- 可选剧名、题材、来源
- 可选目标输出类型

#### 输出

- `job_id`
- 任务记录
- 原始文件路径
- 接入元数据

#### 失败兜底

- 不支持格式：拒绝创建任务
- 文件损坏：标记 `failed_ingest`
- 时长过短：提示不是整剧母版

#### 人工介入点

- 手动补录剧名和题材
- 手动确认是否为完整母版
- 手动指定输出偏好

#### 开发重点

- 校验文件存在与基础可读性
- 生成统一任务目录
- 创建数据库任务记录

### 7.2 环节二：视频标准化

#### 输入

- 原始视频路径
- 目标标准化配置

#### 输出

- `normalized.mp4`
- `video_meta.json`

#### 失败兜底

- 转码失败：自动重试一次
- 转码仍失败：允许保留原片进入下一步，但做风险标记

#### 人工介入点

- 允许手动跳过标准化
- 允许替换坏源文件

#### 开发重点

- 统一编码格式
- 提取时长、fps、分辨率、音轨信息
- 保存标准化日志

### 7.3 环节三：基础解析

解析层建议拆成四个子任务并行或半并行执行。

#### 输入

- 标准化视频
- 可选外挂字幕

#### 输出

- `utterances.json`
- `shots.json`
- `audio_features.json`
- `ocr_events.json`

#### 失败兜底

- ASR 失败：若有外挂字幕则改用外挂字幕
- 镜头切分失败：降级为固定时间窗口切段
- OCR 失败：不阻断流程，仅标记缺省
- 音频特征失败：关闭对应评分项

#### 人工介入点

- 手动上传字幕
- 手动忽略某段异常音频
- 抽查解析质量

#### 开发重点

- ASR 输出句级时间戳
- 镜头切分保持不过碎不过粗
- 音频特征以“剪辑可用”为目标，而非学术准确率

### 7.4 环节四：剧情段切分与剧情理解

#### 输入

- 台词时间轴
- 镜头切分结果
- 音频特征
- OCR 信息

#### 输出

- `story_segments.json`
- 段摘要
- 段标签
- 段角色
- 主题线
- 高光点

#### 失败兜底

- 摘要质量差：回退到规则摘要
- 主题线不稳定：先只保留主线与高光点
- 分段异常：重新合并或再切一次

#### 人工介入点

- 手动改段摘要
- 手动合并/拆分段
- 手动指定主题线
- 手动锁定“必用段”与“禁用段”

#### 开发重点

- 先粗分段，再理解，不要一开始就追求精细段
- 输出结果必须可读、可看、可改
- 每段至少具备摘要、主题、结构作用、情绪类型

### 7.5 环节五：评分与候选池生成

#### 输入

- 剧情段
- 标签
- 主题线
- 高光点
- 音视频特征

#### 输出

- 主力候选段池
- 辅助候选段池
- 各类评分结果

#### 失败兜底

- 评分不稳定：退回规则权重
- 高光点过少：放宽阈值
- 候选池过度集中：添加多样性惩罚

#### 人工介入点

- 手动提高某类情绪权重
- 手动锁定某些段
- 手动剔除不适合商用片段

#### 开发重点

- 评分拆解为多个独立维度
- 单独引入 `suspense_exit_score`
- 对重复事件和重复台词建立惩罚项

### 7.6 环节六：自动编排与时间轴生成

#### 输入

- 候选段池
- 模板类型
- 目标时长
- 去重约束

#### 输出

- 主力 A 时间轴
- 主力 B 时间轴
- 辅助 1 到 6 时间轴
- 桥接文案
- `timeline.json`

#### 失败兜底

- 主力不连贯：回退到主线优先模板
- 主力 A/B 过度重复：强制更换主题线
- 辅助素材过度相似：重跑多样性筛选
- 时长超限：优先压背景段

#### 人工介入点

- 换开头钩子
- 换结尾断点
- 替换某个片段
- 选择更完整或更刺激的版本

#### 开发重点

- 模板驱动优先
- 段内压缩与段间桥接是必须功能
- 编排器必须支持多次重跑而不重做解析

### 7.7 环节七：导出、质检与人工快审

#### 输入

- 时间轴
- 原片路径
- 字幕与包装配置

#### 输出

- 主力素材 MP4
- 辅助素材 MP4
- 字幕文件
- 导出日志
- 封面建议和标题建议

#### 失败兜底

- 导出失败：自动重试一次
- 某条导出失败：不影响其他条继续导出
- 字幕错位：先导出无字幕版
- 音视频不同步：回退上一版时间轴

#### 人工介入点

- 最终快审
- 删除不满意版本
- 替换 CTA、标题、封面
- 选择单条重导出

#### 开发重点

- 时间轴导出稳定性优先于视觉包装
- 先做纯切片拼接，再逐步叠加字幕和包装

## 8. 主力素材模板设计

### 8.1 主力素材 A：主线承接版

目标：

- 让观众看懂主要矛盾
- 建立代入
- 进入上头状态
- 最终卡在高转化断点

结构：

- `0:00 - 0:20` 开场钩子
- `0:20 - 1:20` 最小背景
- `1:20 - 3:00` 第一轮冲突
- `3:00 - 5:00` 反转升级
- `5:00 - 7:20` 高潮临界
- `7:20 - 8:00` 悬念断点

硬约束：

- 前 5 秒必须出现强冲突或强情绪
- 前 90 秒必须说明主要矛盾
- 前 3 分钟必须完成站队
- 结尾必须未闭环

### 8.2 主力素材 B：主题强化版

目标：

- 围绕单一主题线拉满情绪与卖点

适合主题：

- 女主逆袭
- 追妻火葬场
- 豪门身份线
- 婆媳大战

结构：

- `0:00 - 0:15` 强钩子
- `0:15 - 0:50` 极简背景
- `0:50 - 2:30` 高强度冲突
- `2:30 - 4:30` 主题反转
- `4:30 - 6:30` 情绪兑现
- `6:30 - 8:00` 悬念断点

差异约束：

- 与主力 A 核心事件重合率低于 50%
- 开头钩子不能相同
- 结尾断点不能相同

## 9. 辅助素材模板设计

建议固定六类：

1. 最强钩子版
2. 最强冲突版
3. 最强反转版
4. 情绪共鸣版
5. 爽感浓缩版
6. 悬念卡点版

每条建议时长为 60 到 90 秒。

生成原则：

- 每类模板至少生成 2 个候选
- 系统从候选中筛出 1 个正式版
- 任意两条辅助素材的核心事件重合率不高于 60%
- 开头 8 秒尽量不同

## 15. 失败兜底总策略

### 15.1 原则

- 失败优先降级，不轻易全任务终止
- 可恢复阶段必须支持从中间继续
- 非核心增强能力失败时，不影响主链路

### 15.2 降级顺序

- ASR 失败 -> 使用外挂字幕
- 镜头切分失败 -> 使用固定窗口切段
- OCR 失败 -> 跳过 OCR
- 主题线识别失败 -> 只保留主线与高光点
- 高级评分失败 -> 使用规则权重
- 导出包装失败 -> 先导出纯净版

## 17. 验收标准

### 17.1 技术验收

- 单部剧任务可完整跑通
- 任意阶段失败后支持恢复
- 中间结果可查询可预览
- 导出成片音画同步

### 17.2 内容验收

- 主力 A 看起来像完整故事
- 主力 B 与 A 有清晰差异
- 辅助 6 条具有不同测试目标
- 至少 4 条辅助素材前 8 秒不同

### 17.3 业务验收

- 一次完整分析可输出 2 条主力 + 6 条辅助
- 人工复核时间在 10 到 20 分钟
- 至少能支持一剧多剪

## 20. 最终结论

这套系统的本质不是“自动剪一条视频”，而是：

**通过一次完整分析建立剧情资产库，再基于模板、评分、去重与断点策略，批量生成一个可运营的素材家族。**

开发时必须坚持两个原则：

- 中间结果必须可见、可改、可重跑
- 重分析和轻编排必须解耦
