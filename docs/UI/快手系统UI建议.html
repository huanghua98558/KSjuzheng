# 项目说明

这是一个 AI 全自动短剧矩阵运营系统的 Windows 商业客户端。

目标：
- 从 Streamlit 技术后台升级为商业化 Windows 软件
- 用户端必须傻瓜式操作
- 普通用户不需要理解 Agent、任务队列、配置、日志
- 复杂功能隐藏到高级模式

## 技术方向

优先使用 PyQt6 开发 Windows 客户端。
现有 Python 后端、SQLite、任务调度、发布流水线、Agent 系统尽量不改。
本阶段只重构 UI 层和必要的数据适配层。

## 重要原则

1. 不允许随意修改核心业务逻辑
2. 不允许修改采集、发布、签名、任务执行等底层逻辑
3. UI 通过 service / adapter 层读取数据
4. 所有危险操作必须二次确认
5. 普通模式只展示用户能理解的内容
6. 高级模式才显示任务队列、配置、Agent、系统开关
7. 所有页面必须支持空状态、加载状态、错误状态
8. 所有按钮必须有明确反馈

## 用户端导航

普通模式：
- 首页总览
- 账号管理
- 爆款雷达
- 发布管理
- 收益分析
- 风控中心
- 订阅中心
- 设置

高级模式：
- 运行记录
- Agents 中枢
- 系统健康
- 系统开关
- 配置中心
- 数据导出
- 日志中心

## 禁止事项

- 不要在普通用户界面暴露 task_id、sig、breaker、recipe、internal config 等技术字段
- 不要让用户直接编辑复杂参数
- 不要把日志页面作为用户主页面
- 不要破坏现有数据库结构
- 不要提交真实 Cookie、Token、MCN 密钥

## 开发方式

每次开发前必须：
1. 先阅读相关文档
2. 输出开发计划
3. 列出将修改的文件
4. 等确认后再改代码
5. 每完成一个页面，给出运行方式和测试点
